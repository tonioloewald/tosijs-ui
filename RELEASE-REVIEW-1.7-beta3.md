# Pre-Release Review ŌĆö tosijs-ui 1.7.0-beta.3 (base `main`, minor bump)

**Verdict: BLOCK ŌĆö 1 confirmed blocker.** Fix the eager `tjs-lang/editors` import (restores the optional-peer contract + removes reader-path AST parsing); after that this is a clean GO-with-followups. A deliberate semver decision and a stale CHANGELOG want resolving before the *final* 1.7.0 tag but are not code defects.

- **dryness** ŌĆö Several benign duplications introduced/left by this release (gzip-size one-liner ├Ś2, spawn-a-bun-child ├Ś3+, executeInline/executeInIframe prep, and the now-dead `demo/src/style.ts` palette copy). None affect behavior; file to TODO.
- **docs** ŌĆö CHANGELOG is stale for the exact version being cut: no `1.7.0-beta.3` section, the documented tjs-lang peer floor (`^0.9.1`) contradicts what ships (`^0.10.1`), and a "Remaining" note describes work that actually landed. Confirmed. Also: the non-deterministic ePub dirties the build-then-diff staleness gate.
- **coverage** ŌĆö CI e2e runs Chromium only; Firefox/WebKit gate the release's headline browser features solely in the manual local lane (the lane CLAUDE.md itself notes "rots silently"). The four new feature specs additionally `test.skip` WebKit unconditionally. No red tests ŌĆö this is a gap, not a failure.
- **dx** ŌĆö Duplicate of the CHANGELOG-docs finding (peer-range mismatch bites a consumer pinning the documented floor). Merged.
- **ecosystem** ŌĆö Open issues #15 (ePub dead cross-links) and #14 (throwing example) ship still-open/partially-fixed; dispositions for #3/#8/#9/#12/#13 need confirming so tjs-lang's closed #10/#12/#15/#16 aren't conflated with this repo's same-numbered open ones. - **blast-radius** ŌĆö Breaking `<tosi-code>` API (ACEŌåÆCodeMirror) ships under a **minor** bump, so every `^1.6` consumer auto-updates into the break. Deliberate + documented (tied to tosijs 2.0 sequencing) ŌĆö a decision to confirm loudly, not a defect.

## BLOCKER (must fix before cutting)

- [ ] **Eager optional-peer import + unconditional scope-capture in the reader fast path** ŌĆö `src/live-example/execution.ts:2`, `src/live-example/component.ts:1584,1598`. *(verified/confirmed)*
  - `import { scopeCaptureEpilogue } from 'tjs-lang/editors'` is the first and only static (non-type) tjs-lang import in shipped `src`; it survives unbundled into `dist/live-example/execution.js:2`. `doc-browser.ts:175` statically imports `./live-example`, and `tjs-lang` is an **optional** peer (`peerDependenciesMeta.optional`, not a hard `dependency`). So an ESM/bundler consumer of `tosijs-ui/doc-browser` or `tosijs-ui/live-example` who omits the peer (e.g. a `js`-only doc site) now hits an **unresolvable bare specifier at bundle time** ŌĆö a hard break where `main` degraded gracefully via lazy import. This violates the documented contract ("a plain component consumer never pulls tjs-lang in").
  - Even when tjs-lang *is* installed, `refresh()` passes `onScope: this.captureScope` for **every** dialect, so `scopeCaptureEpilogue()` (an acorn AST parse) runs on every auto-run example on page load ŌĆö including plain `js` and readers who never open an editor ŌĆö yet `capturedScope` is consumed only by tjs autocomplete (`liveBindings()`, gated `dialect!=='js' && editorsBuilt`). Pure waste on the common reader path.
  - **Fix:** (1) load `scopeCaptureEpilogue` via a dynamic `import('tjs-lang/editors')` inside `withScopeCapture` (mirroring how `code-editor-cm.ts` already lazy-loads it), keeping tjs-lang out of the static graph; (2) pass `onScope` only when `dialect !== 'js'` (and ideally only once an editor panel is built), matching the `ensureProductTabs()` guard. This folds in the standalone minor finding on `onScope` retention (DOM/typed-array pinning on js examples).

## DECISION POINT (confirm before final 1.7.0)

- [ ] **Breaking `<tosi-code>` API ships under a MINOR bump** ŌĆö `src/code-editor.ts:208` *(blast-radius, verified/confirmed)*. `^1.6.0` consumers auto-resolve 1.7.0; `theme`/`options`/`ace` warn-once-and-no-op, and `editor.session` throws a bare `TypeError` (uncatchable ŌĆö `editor` returns a real CM `EditorView`). Not a defect ŌĆö but the release manager should explicitly ratify shipping breaking changes as a minor, and if kept, make the semver deviation loud at the top of CHANGELOG + README. (Consider 2.0.0 if the sequencing constraint allows.)
